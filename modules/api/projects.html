Projects API¶

List projects¶

List all projects available on the server.

GET /projects

An example could look as follows:

curl -X GET \
  http://dtf.example.com/api/projects

which might give a result like this:

[
    {
        "id": 1,
        "name": "Demo Project",
        "slug": "demo",
        "created": "2021-04-06T07:21:22.515000Z",
        "last_updated": "2021-04-06T07:21:22.515000Z",
        "url": "http://dtf.example.com/demo"
    }
]

Create new project¶

Create a new, global project on the server.

POST /projects

Attribute

Type

Required

Description

name

string

yes

The name of the project to create.

slug

string

yes

Slug for the project to be used in the URL. This needs to be a valid slug and it needs to be unique over all projects.

An example could look as follows:

curl -X POST \
  --header "Content-Type: application/json" \
  --data '{
             "name": "My Test Project",
             "slug": "my-test-project"
         }' \
  http://dtf.example.com/api/projects

which might give a result like this:

{
    "id": 2,
    "name": "My Test Project",
    "slug": "my-test-project",
    "created": "2021-04-06T14:03:49.164368Z",
    "last_updated": "2021-04-06T14:03:49.165367Z",
    "url": "http://dtf.example.com/my-test-project"
}

Get single project¶

Retrieve a specific project from the server.

GET /projects/:id

Attribute

Type

Required

Description

:id

string/integer

yes

The id or slug of the project to get.

An example using the id could look as follows:

curl -X GET \
  http://dtf.example.com/api/projects/1

or using the project slug:

curl -X GET \
  http://dtf.example.com/api/projects/demo

which might give a result like this:

{
    "id": 1,
    "name": "Demo Project",
    "slug": "demo",
    "created": "2021-04-06T07:21:22.515000Z",
    "last_updated": "2021-04-06T07:21:22.515000Z",
    "url": "http://dtf.example.com/demo"
}

Modify single project¶

Modify the fields of an existing project. All fields have to be given (even the ones that are unchanged).

PUT /projects/:id

Attribute

Type

Required

Description

:id

string/integer

yes

The id or slug of the project to modify.

name

string

yes

The new name of the project.

slug

string

yes

The new slug of the project. This needs to be a valid slug and it needs to be unique over all projects.

An example could look as follows:

curl -X PUT \
  --header "Content-Type: application/json" \
  --data '{
             "name": "Demo Project New Name",
             "slug": "demo-project"
         }' \
  http://dtf.example.com/api/projects/1

which might give a result like this:

{
    "id": 1,
    "name": "Demo Project New Name",
    "slug": "demo-project",
    "created": "2021-04-06T07:21:22.515000Z",
    "last_updated": "2021-04-06T14:03:49.131395Z",
    "url": "http://dtf.example.com/demo-project"
}

Add new project property¶

Add a new property to a project.

POST /projects/:project_id/properties

Attribute

Type

Required

Description

:project_id

string/integer

yes

The id or slug of the project to add the new property to.

name

string

no (default: false)

The name of the property to create.

required

boolean

no (default: false)

Whether the property is required for new submissions.

display

boolean

no (default: false)

Whether the display a column for this property in the submissions table.

display_as_link

boolean

no (default: false)

Whether the display this property as link in the submissions table.

display_replace

string

no (default: "")

Instead of the value of this property, this text will be displayed. Use {VALUE} as a placeholder for the original value.

influence_reference

boolean

no (default: false)

Include values for this property to find the references of a submission.

An example could look as follows:

curl -X POST \
  --header "Content-Type: application/json" \
  --data '{
             "name": "Property Name",
             "required": false,
             "display": true,
             "display_as_link": false,
             "display_replace": "Value is {VALUE}",
             "influence_reference": false
         }' \
  http://dtf.example.com/api/projects/1/properties

which might give a result like this:

{
    "project": 1,
    "id": 4,
    "name": "Property Name",
    "required": false,
    "display": true,
    "display_replace": "Value is {VALUE}",
    "display_as_link": false,
    "influence_reference": false
}

Add new project member¶

Add a new member to a project.

POST /projects/:project_id/members

Attribute

Type

Required

Description

:project_id

string/integer

yes

The id or slug of the project to add the new member to.

user

integer

yes

The user to add as a member to the project.

role

string

no (default: "guest")

The role of the user to be added. Can be ‘guest’, ‘developer’ or ‘owner’.

An example could look as follows:

curl -X POST \
  --header "Content-Type: application/json" \
  --data '{
             "user": 1,
             "role": "developer"
         }' \
  http://dtf.example.com/api/projects/1/members

which might give a result like this:

{
    "id": 2,
    "project": 1,
    "user": {
        "id": 1,
        "username": "root",
        "first_name": "",
        "last_name": ""
    },
    "role": "developer"
}
