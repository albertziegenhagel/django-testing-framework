Projects API¶

List projects¶

List all projects available on the server.

GET /projects

An example could look as follows:

curl -X GET \
  http://dtf.example.com/api/projects

curl -X GET `
  http://dtf.example.com/api/projects

which might give a result like this:

[
    {
        "id": 1,
        "name": "Demo Project",
        "slug": "demo",
        "created": "2021-04-06T07:21:22.515000Z",
        "last_updated": "2021-04-06T07:21:22.515000Z",
        "url": "http://dtf.example.com/demo"
    }
]

Create new project¶

Create a new, global project on the server.

POST /projects

Attribute

Type

Required

Description

name

string

yes

The name of the project to create.

slug

string

yes

Slug for the project to be used in the URL. This needs to be a valid slug and it needs to be unique over all projects.

An example could look as follows:

curl -X POST \
  --header "Content-Type: application/json" \
  --data '{
             "name": "My Test Project",
             "slug": "my-test-project"
         }' \
  http://dtf.example.com/api/projects

curl -X POST `
  --header "Content-Type: application/json" `
  --data '{
             ""name"": ""My Test Project"",
             ""slug"": ""my-test-project""
         }' `
  http://dtf.example.com/api/projects

which might give a result like this:

{
    "id": 2,
    "name": "My Test Project",
    "slug": "my-test-project",
    "created": "2021-04-06T14:03:49.164368Z",
    "last_updated": "2021-04-06T14:03:49.165367Z",
    "url": "http://dtf.example.com/my-test-project"
}

Add new project property¶

Add a new property to a project.

POST /projects/:project_id/properties

Attribute

Type

Required

Description

:project_id

string/integer

yes

The id or slug of the project to add the new property to.

name

string

no (default: false)

The name of the property to create.

required

boolean

no (default: false)

Whether the property is required for new submissions.

display

boolean

no (default: false)

Whether the display a column for this property in the submissions table.

display_as_link

boolean

no (default: false)

Whether the display this property as link in the submissions table.

display_replace

string

no (default: "")

Instead of the value of this property, this text will be displayed. Use {VALUE} as a placeholder for the original value.

influence_reference

boolean

no (default: false)

Include values for this property to find the references of a submission.

An example could look as follows:

curl -X POST \
  --header "Content-Type: application/json" \
  --data '{
             "name": "Property Name",
             "required": false,
             "display": true,
             "display_as_link": false,
             "display_replace": "Value is {VALUE}",
             "influence_reference": false
         }' \
  http://dtf.example.com/api/projects/1/properties

curl -X POST `
  --header "Content-Type: application/json" `
  --data '{
             ""name"": ""Property Name"",
             ""required"": false,
             ""display"": true,
             ""display_as_link"": false,
             ""display_replace"": ""Value is {VALUE}"",
             ""influence_reference"": false
         }' `
  http://dtf.example.com/api/projects/1/properties

which might give a result like this:

{
    "project": 1,
    "id": 4,
    "name": "Property Name",
    "required": false,
    "display": true,
    "display_replace": "Value is {VALUE}",
    "display_as_link": false,
    "influence_reference": false
}

Required

Description

Add new project webhook¶

Add a new webhook to a project.

POST /projects/:project_id/webhooks

Attribute

Type

Required

Description

:project_id

string/integer

yes

The id or slug of the project to add the new webhook to.

name

string

yes

The name of the new webhook.

url

url

yes

The URL to be triggered by the webhook.

secret_token

string

yes

This is send in the request header as X-DTF-Token and should be used by the client to authenticate the payload.

on_submission

boolean

no (default: true)

Trigger when a submission is created, modified or deleted.

on_test_result

boolean

no (default: true)

Trigger when test results for a submission are created, modified or deleted.

on_reference_set

boolean

no (default: true)

Trigger when a reference set is created, modified or deleted.

on_test_reference

boolean

no (default: true)

Trigger when test references are created, modified or deleted.

An example could look as follows:

curl -X POST \
  --header "Content-Type: application/json" \
  --data '{
             "name": "My Webhook",
             "url": "http://example.com/my/hook/path",
             "secret_token": "sMeyqOYenB8Czi2zGOFHNg",
             "on_submission": false
         }' \
  http://dtf.example.com/api/projects/1/webhooks

curl -X POST `
  --header "Content-Type: application/json" `
  --data '{
             ""name"": ""My Webhook"",
             ""url"": ""http://example.com/my/hook/path"",
             ""secret_token"": ""sMeyqOYenB8Czi2zGOFHNg"",
             ""on_submission"": false
         }' `
  http://dtf.example.com/api/projects/1/webhooks

which might give a result like this:

{
    "project": 1,
    "id": 2,
    "name": "My Webhook",
    "url": "http://example.com/my/hook/path",
    "secret_token": "sMeyqOYenB8Czi2zGOFHNg",
    "on_submission": false,
