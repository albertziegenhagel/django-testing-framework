References API¶

List reference sets of a project¶

List all reference sets of a given project.

GET /projects/:project_id/references

Attribute

Type

Required

Description

:project_id

string/integer

yes

The id or slug of the project to get the reference sets of.

An example could look as follows:

curl -X GET \
  http://dtf.example.com/api/projects/1/references

curl -X GET `
  http://dtf.example.com/api/projects/1/references

which might give a result like this:

[
    {
        "project": 1,
        "id": 1,
        "created": "2021-04-06T10:03:29.011000+02:00",
        "last_updated": "2021-04-06T10:03:29.011000+02:00",
        "property_values": {
            "Branch": "main",
            "Hostname": "Vasa"
        }
    }
]

Create new reference set¶

Create a new, empty reference set for the project.

POST /projects/:project_id/references

Attribute

Type

Required

Description

:project_id

string/integer

yes

The id or slug of the project to add the new reference set to.

property_values

json

no (default: {})

Property value pairs for the reference set. These have to match the properties specified in the project.

An example could look as follows:

curl -X POST \
  --header "Content-Type: application/json" \
  --data '{
             "property_values": {
                 "Hostname": "Vasa",
                 "Branch": "feature"
             }
         }' \
  http://dtf.example.com/api/projects/1/references

curl -X POST `
  --header "Content-Type: application/json" `
  --data '{
             ""property_values"": {
                 ""Hostname"": ""Vasa"",
                 ""Branch"": ""feature""
             }
         }' `
  http://dtf.example.com/api/projects/1/references

which might give a result like this:

{
    "project": 1,
    "id": 2,
    "created": "2022-02-15T14:49:01.772663+01:00",
    "last_updated": "2022-02-15T14:49:01.772663+01:00",
    "property_values": {
        "Branch": "feature",
        "Hostname": "Vasa"
    }
}

Modify reference sets of a project¶

Modify the fields of an existing reference set. All fields have to be given (even the ones that are unchanged).

PUT /projects/:project_id/references/:reference_id

Attribute

Type

Required

Description

:project_id

string/integer

yes

The id or slug of the project.

:reference_id

integer

yes

The id of the reference set to modify.

property_values

json

no (default: {})

Property name-value pairs for the reference set. These have to match the properties specified in the project.

An example could look as follows:

curl -X PUT \
  --header "Content-Type: application/json" \
  --data '{
             "project": 1,
             "property_values": {
                 "Hostname": "Vasa",
                 "Branch": "main"
             }
         }' \
  http://dtf.example.com/api/projects/1/references/1

curl -X PUT `
  --header "Content-Type: application/json" `
  --data '{
             ""project"": 1,
             ""property_values"": {
                 ""Hostname"": ""Vasa"",
                 ""Branch"": ""main""
             }
         }' `
  http://dtf.example.com/api/projects/1/references/1

which might give a result like this:

{
    "project": 1,
    "id": 1,
    "created": "2021-04-06T10:03:29.011000+02:00",
    "last_updated": "2022-02-15T14:49:00.604396+01:00",
    "property_values": {
        "Branch": "main",
        "Hostname": "Vasa"
    }
}

Delete reference sets of a project¶

Deletes a reference set and all its associated data (like test result references) from a project. This can not be undone!

DELETE /projects/:project_id/references/:reference_id

Attribute

Type

Required

Description

:project_id

string/integer

yes

The id or slug of the project.

:reference_id

integer

yes

The id of the reference set to delete.

An example could look as follows:

curl -X DELETE \
  http://dtf.example.com/api/projects/1/references/2

curl -X DELETE `
  http://dtf.example.com/api/projects/1/references/2
