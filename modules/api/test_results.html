Rest Results API¶

List test results of a project¶

GET /projects/:project_id/tests

Attribute

Type

Required

Description

:project_id

string/integer

yes

The id or slug of the project to get the test results of.

curl -X GET \
  http://dtf.example.com/api/projects/1/tests

Get single test result of a project¶

GET /projects/:project_id/tests/:test_id

Type

Required

Description

string/integer

yes

The id or slug of the project.

:test_id

integer

yes

The id of the test results to get.

curl -X GET \
  http://dtf.example.com/api/projects/1/tests/1

Modify test results of a project¶

Modify the fields of an existing test result. All fields have to be given (even the ones that are unchanged).

PUT /projects/:project_id/tests/:test_id

Attribute

Type

Required

Description

:project_id

string/integer

yes

The id or slug of the project.

:test_id

integer

yes

The id of the test results to modify.

name

string

yes

The new name of the test.

status

string

yes

Status of the test. Allowed values are ‘skip’, ‘successful’, ‘unstable’, ‘unknown’, ‘failed’, or ‘broken’.

results

json

yes

Test results. TODO: add JSON schema

Delete test results of a project¶

Deletes a test result from a project. This can not be undone!

DELETE /projects/:project_id/tests/:test_id

Attribute

Type

Required

Description

:project_id

string/integer

yes

The id or slug of the project.

:test_id

integer

yes

The id of the test results to delete.

An example could look as follows:

curl -X DELETE \
  http://dtf.example.com/api/projects/1/tests/1

curl -X DELETE `
  http://dtf.example.com/api/projects/1/tests/1

History of a test result measurement¶

Retrieves the history of a measurement for a given test. The history is extracted for the all test results of that project that belong to a submission which has the same reference relevant properties as the submission the current test belongs to.

GET /projects/:project_id/submissions/:submission_id/tests/:test_id/history

Attribute

Type

Required

Description

:project_id

string/integer

yes

The id or slug of the project.

:submission_id

integer

yes

The id of the submission the test result belongs to.

:test_id

integer

yes

The id of the test result to get the history for.

measurement_name

integer

yes

Name of the measurement to get the history for.

limit

integer

no

Maximum number of entries to be retrieved. Counting is started from the most recent entry to older ones.

An example could look as follows:

curl -X GET \
  http://dtf.example.com/api/projects/1/submissions/1/tests/1/history?measurement_name=runtime&limit=2
