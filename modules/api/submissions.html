Submissions API

List submissions of a project

List all submissions of a given project.

GET /projects/:project_id/submissions

Attribute

Type

Required

Description

:project_id

string/integer

yes

The id or slug of the project to get the submissions of.

An example could look as follows:

curl -X GET \
  http://dtf.example.com/api/projects/1/submissions

curl -X GET `
  http://dtf.example.com/api/projects/1/submissions

Create new submission

Create a new, empty submission for the project.

POST /projects/:project_id/submissions

Attribute

Type

Required

Description

:project_id

string/integer

yes

The id or slug of the project to add the new submission to.

info

json

no (default: {})

Property value pairs for the submission. These have to match the properties specified in the project.

An example could look as follows:

curl -X POST \
  --header "Content-Type: application/json" \
  --data '{
             "info": {
                 "Hostname": "Vasa",
                 "Branch": "feature"
             }
         }' \
  http://dtf.example.com/api/projects/1/submissions

curl -X POST `
  --header "Content-Type: application/json" `
  --data '{
             ""info"": {
                 ""Hostname"": ""Vasa"",
                 ""Branch"": ""feature""
             }
         }' `
  http://dtf.example.com/api/projects/1/submissions

which might give a result like this:

{
    "project": 1,
    "id": 3,
    "created": "2022-02-15T13:10:02.723719+01:00",
    "last_updated": "2022-02-15T13:10:02.723719+01:00",
    "info": {
        "Hostname": "Vasa",
        "Branch": "feature"
    },
    "url": "http://dtf.example.com/demo-project/submissions/3"
}

Modify submissions of a project

Modify the fields of an existing submission. All fields have to be given (even the ones that are unchanged).

PUT /projects/:project_id/submissions/:submission_id

Attribute

Type

Required

Description

:project_id

string/integer

yes

The id or slug of the project.

:submission_id

integer

yes

The id of the submission to modify.

info

json

no (default: {})

Property name-value pairs for the submission. These have to match the properties specified in the project.

An example could look as follows:

curl -X PUT \
  --header "Content-Type: application/json" \
  --data '{
             "project": 1,
             "info": {
                 "Hostname": "Vasa",
                 "Branch": "feature"
             }
         }' \
  http://dtf.example.com/api/projects/1/submissions/2

curl -X PUT `
  --header "Content-Type: application/json" `
  --data '{
             ""project"": 1,
             ""info"": {
                 ""Hostname"": ""Vasa"",
                 ""Branch"": ""feature""
             }
         }' `
  http://dtf.example.com/api/projects/1/submissions/2

which might give a result like this:

{
    "project": 1,
    "id": 2,
    "created": "2021-04-06T10:10:24.740000+02:00",
    "last_updated": "2022-02-15T13:10:01.207524+01:00",
    "info": {
        "Hostname": "Vasa",
        "Branch": "feature"
    },
    "url": "http://dtf.example.com/demo/submissions/2"
}

Delete submissions of a project

Deletes a submission and all its associated data (like test results) from a project. This can not be undone!

DELETE /projects/:project_id/submissions/:submission_id

Attribute

Type

Required

Description

:project_id

string/integer

yes

The id or slug of the project.

:submission_id

integer

yes

The id of the submission to delete.

An example could look as follows:

curl -X DELETE \
  http://dtf.example.com/api/projects/1/submissions/3

curl -X DELETE `
  http://dtf.example.com/api/projects/1/submissions/3
